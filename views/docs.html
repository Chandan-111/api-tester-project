Dashboard

Welcome to the API Learning Server Documentation. The base URL is your server's root URL (e.g., http://localhost:3000). All requests and responses use JSON.

Available Resources

You can use any of these resource paths for the methods below:
/users, /products, /orders, /employees, /books, /students, /tasks

GET /[resource]

Description: Retrieve a list of records. Supports pagination, sorting, and filtering.

Query Parameters:
?_limit=10 (Max records to return)
?_page=1 (Page number)
?_sort=id&_order=desc (Sorting)

Example Request (JavaScript Fetch)

fetch('http://localhost:3000/users?_limit=2')
  .then(res => res.json())
  .then(console.log);

Example Response JSON

{
  "success": true,
  "message": "users fetched successfully",
  "data": [
    {
      "id": 1,
      "name": "John Doe",
      "email": "john.doe@example.com",
      "created_at": "2023-10-25T12:00:00Z"
    },
    {
      "id": 2,
      "name": "Jane Smith",
      "email": "jane.smith@example.com",
      "created_at": "2023-10-25T12:05:00Z"
    }
  ],
  "meta": { "total": 15, "page": 1, "limit": 2 }
}
GET /[resource]/:id

Description: Retrieve a single record by its ID.

Example Request (cURL)

curl http://localhost:3000/products/5

Example Response JSON

{
  "success": true,
  "message": "products fetched successfully",
  "data": {
    "id": 5,
    "name": "USB-C Hub",
    "price": 29.99,
    "description": "7-in-1 port expansion",
    "created_at": "2023-10-25T12:00:00Z"
  }
}
POST /[resource]

Description: Create a new record.

Request Body (JSON)

{
  "name": "Alex Johnson",
  "email": "alex.j@example.com"
}

Example Request (JavaScript Fetch)

fetch('http://localhost:3000/users', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ 
    name: 'Alex Johnson', 
    email: 'alex.j@example.com' 
  })
})
.then(res => res.json())
.then(console.log);

Example Response JSON (201 Created)

{
  "success": true,
  "message": "users created successfully",
  "data": {
    "id": 16,
    "name": "Alex Johnson",
    "email": "alex.j@example.com",
    "created_at": "2023-10-26T08:30:00Z"
  }
}
PUT /[resource]/:id

Description: Replace/Update an entire record. (In this API, PUT and PATCH behave similarly).

Request Body (JSON)

{
  "title": "Clean the garage and basement",
  "completed": true
}

Example Request (Power Automate / HTTP Action)

Method: PUT
URI: http://localhost:3000/tasks/8
Headers: { "Content-Type": "application/json" }
Body: 
{
  "title": "Clean the garage and basement",
  "completed": true
}

Example Response JSON

{
  "success": true,
  "message": "tasks updated successfully",
  "data": {
    "id": 8,
    "title": "Clean the garage and basement",
    "completed": true,
    "created_at": "2023-10-25T12:00:00Z"
  }
}
PATCH /[resource]/:id

Description: Partially update a record. Only send the fields you want to change.

Request Body (JSON)

{
  "price": 899.99
}

Example Request (Axios)

axios.patch('http://localhost:3000/products/6', {
  price: 899.99
}).then(response => console.log(response.data));

Example Response JSON

{
  "success": true,
  "message": "products updated successfully",
  "data": {
    "id": 6,
    "name": "Smartphone",
    "price": 899.99,
    "description": "Latest model smartphone",
    "created_at": "2023-10-25T12:00:00Z"
  }
}
DELETE /[resource]/:id

Description: Delete a record by ID.

Example Request (Python Requests)

import requests
response = requests.delete('http://localhost:3000/books/15')
print(response.status_code)

Example Response

Status: 204 No Content
(Body is empty)

Utility & Error Endpoints

GET /health

Check server health and database connection.

GET /error/500

Simulate a 500 Internal Server Error for testing error handling in your client. You can replace 500 with any common status code (400, 401, 403, 404, 409, 422, etc.).

Example Response JSON

{
  "success": false,
  "message": "Internal Server Error",
  "errors": [
    "Simulated 500 error"
  ]
}
